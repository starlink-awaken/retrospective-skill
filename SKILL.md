# Retrospective

**复盘工具**：快速回顾项目/任务，提炼经验教训，更新知识库。

## 使用方式

```
/retrospective "你的项目描述或任务结果"
```

## 流程

```
1. 输入 → 2. 分析 → 3. 提炼 → 4. 存储
```

## 输出

- 经验教训 (3-5 条)
- 改进建议 (3-5 条)
- 知识沉淀 (更新到 MEMORY)

## 示例

```
/retrospective "完成了 Context Compression 项目的分析"
↓
输出:
├── 核心教训: 没问过用户 = 猜
├── 关键教训: ROI 保守主义
├── 改进建议: 用户调研前置
└── 知识库: 已更新
```

## 触发词

- "复盘"
- "回顾"
- "总结"
- "提取教训"
