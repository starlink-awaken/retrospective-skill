# Quick Retrospective Workflow

**类型：** Quick (快速复盘)
**耗时：** 5分钟
**适用：** 小任务、日常迭代、bug修复

---

## 📋 Workflow Steps

### Step 1: 触发识别（自动）

**检测关键词：**
- "做完了"、"完成了"、"搞定"
- "bug修复了"、"功能上线了"
- "今天的工作总结"

**用户确认：**
```
PAI：检测到任务完成，是否启动快速复盘（5分钟）？
选项：[是] [否] [稍后]
```

### Step 2: 快速分析（2分钟）

**Agent：** `quick_analyzer`

**分析内容：**
1. **任务概览** - 做了什么？
2. **关键结果** - 达成什么？
3. **主要挑战** - 遇到什么问题？
4. **解决过程** - 如何解决的？
5. **时间投入** - 花了多久？

**输出：**
- 1个经验教训
- 1个改进建议

### Step 3: 关键提取（1分钟）

**提取内容：**
- **成功要素** - 做对的是什么？
- **失败原因** - 哪里可以改进？
- **可复用知识** - 下次能用什么？

**格式：**
```markdown
## 经验教训
[具体教训]

## 改进建议
[具体建议]

## 可复用
[下次直接用的]
```

### Step 4: 记录到MEMORY（2分钟）

**保存位置：**
```
~/.claude/MEMORY/LEARNING/RETROSPECTIVE/2026-02/quick/
└── YYYY-MM-DD-{task_name}.md
```

**记录格式：**
```markdown
# 快速复盘：{task_name}

**时间：** {timestamp}
**类型：** Quick
**耗时：** 5分钟

## 任务概览
- **任务：** {description}
- **结果：** {outcome}
- **时长：** {duration}

## 经验教训
{lesson}

## 改进建议
{suggestion}

## 可复用知识
{knowledge}
```

---

## 🎯 成功标准

- ✅ 5分钟内完成
- ✅ 输出1个经验教训
- ✅ 记录到MEMORY
- ✅ 不阻塞工作流

---

## 📊 示例

### 输入
```
用户："npm install的问题终于解决了"
```

### 执行
```
PAI：检测到问题解决，启动快速复盘...

[quick_analyzer分析2分钟]

✅ 分析完成！

经验教训：依赖问题需要先检查package.json
改进建议：下次先npm ls检查依赖
可复用：将此加入调试SOP

[记录到MEMORY]
```

### 输出
```markdown
# 快速复盘：npm install依赖问题

**时间：** 2026-02-03 16:30
**类型：** Quick

## 任务概览
- **任务：** 解决npm install失败
- **结果：** 成功安装
- **时长：** 30分钟

## 经验教训
依赖问题不应该重复尝试npm install，应该先检查依赖清单

## 改进建议
在调试SOP中增加：npm install失败 → 检查package.json → npm ls

## 可复用知识
失败模式：重复操作相同命令不解决问题
解决模式：先诊断后行动
```
